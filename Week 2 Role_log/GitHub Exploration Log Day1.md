# GitHub Exploration Log
## 参考开源仓库
1. Moss：https://github.com/nishuzumi/moss
2. Argot Solidity：https://github.com/argotorg/solidity
3. Hardhat：https://github.com/NomicFoundation/hardhat

## 一、通用项目目录结构（三个Web3项目共性）
1. 仓库根目录
- `README.md`：项目定位、快速启动命令、核心能力一览，是新手第一入口
- `LICENSE`、`CONTRIBUTING.md`：开源协议、外部开发者贡献规范、PR提交标准
- `SECURITY.md`：漏洞上报渠道、高危操作边界、安全风险提示
- `.github` 文件夹：项目自动化与协作规范配置
  - workflows：CI/CD流水线，自动构建、类型检查、单元测试
  - ISSUE_TEMPLATE / PULL_REQUEST_TEMPLATE：标准化提交模板，规范提问与代码提交格式
2. `/docs`
存放完整教程、API契约、架构设计、协议接入、安全规则等深度文档
3. 源码分包目录
- Moss、Hardhat采用monorepo多包架构，按功能拆分独立package；
- Argot Solidity按编译器职能分层，区分语法解析、ABI生成底层模块
4. `/examples`
可直接运行的最小Demo，降低新手实操门槛

## 二、GitHub 核心模块作用
1. README：项目简介门面，快速看懂项目用途、安装流程、支持协议
2. Docs：系统化完整文档，解决进阶开发、自定义扩展、底层原理疑问
3. Issues：用于提交Bug、功能需求、文档优化诉求，维护者同步排期跟进
4. Pull Requests：开发者提交代码/文档修改，维护者结合自动化测试人工审核后合并
5. Discussions：非代码类交流区，技术答疑、生态方案、开发思路探讨，和Bug需求做分区隔离

## 三、感兴趣的真实 Issue(来自Moss)
### Issue 完整基础信息
编号：#77
标题：`[feature] docs: suggest adding a FAQ document for Chinese newcomers`
标签：enhancement
创建作者：bky0211

### Issue 核心诉求
提交者作为Web3新手初次体验Moss时，发现入门阶段存在大量重复高频疑问，但现有文档没有统一汇总入口，中文开发者需要分散查阅多处资料，拉高入门门槛。因此提议在仓库新建 `docs/faq-zh.md` 中文FAQ文档，集中统一解答国内新手共性疑问，减少社群重复提问，降低中文使用者学习成本。
提交人明确规划了8类核心问答方向，覆盖基础概念、环境前置要求、资金安全、支持链、协议新增、安全机制、模拟告警、框架差异化对比。

### 对应项目开发约束
1. 改动仅存在于`docs`文档文件夹，不修改core、simulator、protocol协议包等底层代码，不会改动项目任何运行逻辑；
2. FAQ所有回答必须与现有`getting-started.zh-CN.md`、`protocol-onboarding.md`、安全说明文档内容完全一致，保证信息不冲突；
3. 文档内所有配置、命令、规则描述均需要本地实操验证，保证内容真实可复现；
4. 提交前完成项目lint、文档格式自动化校验，符合仓库文档提交规范。

### 落地验收标准
1. 新建文件 `docs/faq-zh.md`，覆盖提交人列出的8大核心问题：
    - Moss基础定义（面向AI Agent+DeFi零基础新手）
    - 运行前置依赖（Node≥22、pnpm、基础命令行能力）
    - 无真实资金测试可行性
    - 当前支持区块链（Monad主网，chain id 143）
    - 自定义协议接入流程
    - “永不签名、永不发送”安全机制通俗解释
    - 模拟器告警处理规范
    - Moss与其他AI Agent框架核心差异
2. 语言通俗易懂，用大白话解释专业安全模型、模拟校验规则，适配国内零基础中文开发者；
3. 文中对应操作点补充官方文档跳转链接，例如协议开发跳转模板包与接入指南；
4. FAQ完成后，在中文入门文档增加跳转入口，引导新手优先查阅FAQ。

### 我感兴趣的原因
我日常使用Windows搭建Monad AI Agent Demo，刚接触Moss时也对项目定位、资金安全规则、链支持范围、模拟告警规则存在大量疑问，需要反复翻阅多篇文档才能理清。该Issue为纯文档贡献任务，无需复杂底层代码开发，门槛低、能直接帮助国内大量新手开发者，同时提交者也表示愿意协同撰写内容。我可以承接这份需求，完善这份中文FAQ文档，完成一次完整规范的开源PR贡献实践。

## 四、我的发现
1. 三款Web3工具项目标准化程度极高，依靠`.github`模板与CI流水线统一协作规范，减少维护成本；
2. Moss仓库Issues区分两类工单：普通开发者提交的文档优化需求（enhancement标签）、维护者内部迭代任务（maintainer-only标签），分工清晰；
3. 安全是Web3开源项目核心约束，均独立设立安全文档，对链上转账、AI自主交易等高风险操作做强制校验；
4. 所有文档类优化Issue都有明确落地标准，要求配套实操步骤、测试校验，保障新手可以顺利复现项目流程。

## 五、学习收获
1. 掌握开源仓库分层逻辑，能快速定位文档、示例、自动化配置文件；
2. 看懂真实仓库Issue分类、标签管理方式，理解维护者区分外部贡献与内部迭代任务的管理思路；
3. 学会从真实开放Issue里筛选贴合自身开发环境、学习方向的贡献任务，清晰完整梳理需求落地标准；
4. 总结Web3开源通用管理思路：自动化管控代码质量、优先保障链上资产安全、配套完整新手落地材料。
